# docs 目录说明

本目录存放项目相关的各类文档，按子目录分类。**除特别说明外，绝大部分为临时文档，仅作短期计划或问题跟踪之用，无长期参考价值。**

---

## 目录结构及模块说明

### `flowcharts/` — 流程说明

存放主循环、启动初始化、自动换标等核心流程的说明文档，用于理解代码运行流程。**相对偏参考性质**，可能随代码变更而过时。

---

### `plans/` — 计划文档（按年月组织）

按 `YYYY-MM` 分子目录（如 `2026-01/`、`2026-02/`），存放单次重构、功能或改动的**实施计划**。

**性质：临时计划文档。** 用于当时执行任务时的指导与记录，任务完成后即失去参考作用，请勿作为当前架构或行为的权威依据。

---

### `issues/` — 问题与方案文档

存放与具体问题、Bug 分析、代码审查结论、系统性修复方案等相关的文档，例如：

- 某次代码审查的发现与建议
- 订单监控、浮点精度、交易日历预热等问题的可行性或修复方案
- 与 issues 编号对应的系统性重构/修复计划

**性质：临时文档。** 仅对应当时的问题与结论，问题关闭或方案落地后无持续参考价值。

---

### `others/` — 其他

存放与项目架构无直接关系的杂项文档（如协作提示词、操作指引等）。**性质：个人/团队协作备忘。**

---

## 使用与维护建议

1. **不要依赖本目录作为唯一参考**  
   架构与行为以当前代码和根目录 README、规范类文档为准。

2. **plans 与 issues 文档视为短期产物**  
   绝大部分为临时计划文档，仅作短期执行和记录，**无长期参考作用**。

3. **流程文档可能过时**  
   `flowcharts/` 下文档在代码大幅变更后需人工核对并更新。
